# PROMPT · 分路差异化评判 + 雷达动效优化

## 需求来源（用户原话归纳）
1. 切换选手时雷达"延伸"动画存在，但希望**字段值达到满点(高分)时，端点出现持续闪烁/脉冲动效**。
2. 切换选手时雷达切换**不顺滑、有卡顿**，需优化为平滑过渡。
3. **不同分路(top/jng/mid/bot/sup)的选手，六维评判标准应差异化**：
   不同定位的数据侧重不同，权重应不同（如辅助看视野/团战，下路看输出/经济，打野看资源/节奏）。
4. **队伍同理**：队伍维度也应有自己的合理评判（区别于选手），保持现有 GSPD/GPR 直读优势。

## 需求拆解与定义

### R1 · 分路差异化权重（核心）
- 为 5 个位置各定义一套 `PLAYER_DIM_FORMULA`，共用同一批二级字段输入，仅权重不同。
- 设计原则（基于电竞常识）：
  | 位置 | 侧重 |
  |---|---|
  | top  | 线上压制、操作、抗压(心态)；团战中等 |
  | jng  | 团战决策(节奏)、运营(视野/资源)、版本；线上弱化 |
  | mid  | 操作、线上、团战；均衡型 |
  | bot  | 操作(输出)、线上、经济占比；视野弱化 |
  | sup  | 团战、运营(视野/控眼)、心态；操作/线上弱化 |
- 权重和必须=1.0；维度的"字段名标注(fields)"也随位置变化（体现差异）。
- LPL 线上压制降级逻辑保留。

### R2 · 队伍专属权重
- team 单独一套权重，突出 GSPD/GPR(运营)、击杀节奏(团战)、胜率(心态)。

### R3 · 满点脉冲动效
- 前端：维度端点 value >= 90 时，该点用持续脉冲(半径/透明度循环)高亮。
- 用 Chart.js 自定义插件 + requestAnimationFrame 实现循环重绘(仅重绘点，不重算图)。
- 阈值 90，脉冲周期约 1.2s，颜色用主体色(蓝)。

### R4 · 平滑切换
- 现状 chart.update() 已复用实例，但默认动画在数据跨度大时仍显跳变。
- 优化：统一 update 动画 easing='easeInOutCubic'，duration 600；
  对 dataset 启用按点 delay 渐次展开(可选)；切换时先淡出旧标签再淡入。
- 关键：脉冲 RAF 循环与 chart 过渡动画不冲突(过渡期间不脉冲，过渡完成后才脉冲)。

## 验收标准
- mid 与 sup 同一名选手数据，六维权重不同 → 形状不同(抽样验证两套权重确实生效)。
- fields 标注随位置不同。
- 第一名(如 Chovy)出现 >=90 的端点 → 页面可见脉冲(代码层校验插件已挂载+RAF运行)。
- 切换选手雷达过渡无突兀跳变(easing 生效)。
- 队伍维度用独立权重，Gen.G 仍居首且 GSPD 直读不变。
