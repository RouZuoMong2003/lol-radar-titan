# PROMPT · 雷达图表现优化需求

## 背景问题（数据实证）
当前维度用 z-score 映射（mean=60, 1σ=15），导致：
- 第一名最高仅 ~85，雷达图"不满"，缺乏顶尖选手应有的张力。
- 部分维度组内跨度过小（如团战决策仅 16 分），雷达近似正圆，区分度低。
- 全局 ≥90 占比仅 0.2%，强项维度顶不到边。

## 优化目标
1. **顶尖应满**：每个 (赛季,位置) 分组里，单维度第一名 → 95~100；综合第一名整体外扩到接近满雷达。
2. **差异化明显**：强项与弱项拉开，雷达呈现清晰的多边形形状而非圆。
3. **保持公平**：仍按 (赛季,位置) 同组相对排名，60 仍≈组内中位，弱项可低至 15~30。
4. **不破坏现有契约**：维度仍 0–100；字段名/公式注释/双端 UI 不变。

## 技术方案
### 方案：分位数拉伸（Percentile Rank + 增益曲线）
替换 `zscore_to_100`：
- 组内计算该值的**百分位排名** p ∈ [0,1]（最强=1.0，最弱=0.0）。
- 映射：`score = lo + (hi - lo) * pow(p, gamma)`
  - `hi=99, lo=20`（弱项允许更低，强项顶到 99）
  - `gamma=0.7`（<1：让中上段更舒展，强者更突出）
- 组内**最强者强制=100**（让第一名维度真正满格）。
- 小样本组(<4人)退化为 min-max 线性拉伸到 [40,95]，避免极端。

### 队伍维度
team_dimensions 也用分组百分位（同赛季全队伍），保证队伍第一名也"满"。

### 综合评分
text_score 公式不变（仍由 6 维均值 + 胜率），但因维度整体上移，第一名分数自然更高。

## 验收标准
- LCK-2026-Cup mid 组：Chovy 至少 3 个维度 ≥ 90，最高=100。
- 团战决策这类窄分布维度，组内跨度从 ~16 提升到 ≥ 40。
- 弱队/弱选手仍有明显内缩（最低维度可 < 35），形成对比。
